# Roguelike Game (C#) — Учебный проект

## Описание проекта

Данный проект выполнен в рамках изучения разработки roguelike-игр на C# по курсу  
**Creating a Roguelike Game in C#** (RogueSharp: https://roguesharp.wordpress.com/).

Основная цель проекта — пройти полный цикл разработки roguelike-игры: от базовой архитектуры и генерации уровней до реализации боевой системы, поведения врагов и расширения механик за пределами курса.

Дополнительно проект рассматривается как часть более крупной системы — Telegram-бота для изучения русского языка. В рамках этого бота планируется внедрение мини-игр, которые помогают пользователям закреплять знания в интерактивной форме. Данный roguelike-проект выступает одной из таких мини-игр, демонстрируя возможность интеграции игровых механик в образовательную платформу.

---

## Процесс разработки

Разработка проекта проходила поэтапно, в соответствии со структурой курса, с постепенным усложнением системы.

### 1. Базовая архитектура проекта

На первом этапе была заложена основа игры:

- создание игрового цикла;
- настройка системы рендеринга консольного интерфейса;
- организация основных классов (игрок, враги, карта, тайлы);
- разделение логики на отдельные модули (сущности, поведение, генерация карты).

Особое внимание уделялось архитектуре, чтобы система оставалась расширяемой для дальнейших механик.

---

### 2. Генерация уровней

Следующим этапом стала реализация генерации подземелий:

- создание процедурной генерации комнат и коридоров;
- проверка проходимости карты;
- размещение игрока и врагов на карте;
- обеспечение связности всех зон уровня.

---

### 3. Система перемещения и взаимодействия

После создания карты была реализована логика перемещения:

- обработка ввода игрока;
- перемещение по клеткам;
- проверка столкновений со стенами;
- взаимодействие с объектами окружения.

---

### 4. Боевая система

Была реализована основа боевой системы:

- атака игрока и врагов;
- расчет урона;
- проверка попадания;
- уничтожение сущностей при смерти.

---

### 5. Искусственный интеллект врагов

Реализована базовая логика поведения врагов:

- движение в сторону игрока;
- атака при сближении;
- простое принятие решений на основе расстояния.

---

### 6. Система способностей

Добавлена система способностей персонажа:

- активация умений;
- различные типы эффектов (урон, лечение, область действия);
- ограничения по использованию.

Были реализованы способности:
- дальние атаки;
- массовые атаки;
- лечение;
- эффекты по области.

---

### 7. Добавленные улучшения

#### Способность заморозки (Freeze)

Была добавлена дополнительная механика:

- временная блокировка действий врагов;
- ограничение их перемещения;
- создание тактического преимущества для игрока.

---

### 8. Полировка и балансировка

Проводилась доработка проекта:

- исправление ошибок генерации;
- балансировка параметров;
- улучшение поведения врагов;
- оптимизация игрового цикла.

---

## Интеграция с Telegram-ботом

Проект также рассматривается как часть более крупной системы — Telegram-бота для изучения русского языка. Основная идея бота заключается в обучении через интерактивные сценарии и мини-игры.

В рамках этой концепции планируется:

- добавление различных мини-игр для закрепления материала;
- использование игровых механик для повышения вовлеченности;
- интеграция отдельных модулей, разработанных в рамках учебных проектов.

Данный roguelike-проект является одной из таких мини-игр. Он демонстрирует подход к созданию автономного игрового модуля, который в дальнейшем может быть встроен в Telegram-бота как отдельный обучающий сценарий.
