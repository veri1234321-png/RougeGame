## Вариативная часть проекта

В рамках вариативной части была разработана отдельная мини-игра в жанре roguelike, реализованная на языке программирования C# (версия .NET 6.0). Разработка выполнялась в среде Microsoft Visual Studio на операционной системе Windows. В качестве основы для реализации игрового процесса использовалась библиотека RogueSharp (https://roguesharp.wordpress.com/), предоставляющая инструменты для создания пошаговых roguelike-игр с процедурной генерацией уровней.

---


### Цели проекта

- разработка Telegram-бота для обучения русскому языку иностранных пользователей;
- повышение эффективности обучения за счёт использования интерактивных и игровых механик;
- реализация дополнительных игровых модулей для геймификации образовательного процесса;
- разработка прототипа roguelike-игры для дальнейшей интеграции в бот.
---

### Задачи проекта

- разработать структуру Telegram-бота на языке Python;
- реализовать образовательные и игровые механики внутри бота;
- создать мини-игру для изучения фонетики русского языка;
- разработать отдельную roguelike-игру на C# с использованием RogueSharp;
- реализовать базовые игровые системы (карта, перемещение, боевая система);
- добавить систему заклинаний как расширение игровой механики;
- подготовить игровые элементы для дальнейшей интеграции в Telegram-бот.

### Процесс разработки roguelike-игры

Разработка игры была выполнена поэтапно с постепенным усложнением функционала и опорой на возможности библиотеки RogueSharp.

#### 1. Изучение предметной области и библиотеки RogueSharp
На первом этапе была изучена специфика жанра roguelike, включая ключевые особенности: пошаговый геймплей, процедурная генерация уровней, ограниченная видимость карты (fog of war) и высокая значимость тактических решений.

Параллельно было проведено изучение библиотеки RogueSharp, её основных сущностей (карта, клетки, сущности, поля зрения), а также принципов работы с ней в C#.

#### 2. Настройка проекта в Visual Studio
Был создан проект на C# (.NET 6.0) в среде Microsoft Visual Studio для Windows. Настроена структура проекта, подключены необходимые зависимости и библиотека RogueSharp. Организована базовая архитектура проекта, разделяющая логику игры на отдельные компоненты (карта, игрок, враги, логика управления).

#### 3. Реализация игровой карты
Следующим этапом была разработка системы игровой карты. Реализована сеточная структура уровня, основанная на клетках. С использованием возможностей RogueSharp была добавлена процедурная генерация подземелий, включающая случайное создание комнат и соединяющих их коридоров.

Каждое новое прохождение игры сопровождается генерацией уникального уровня, что повышает реиграбельность проекта.

#### 4. Реализация игрока и базового перемещения
Был добавлен игровой персонаж (игрок), реализовано управление его перемещением по клеткам карты. Движение осуществляется пошагово, с проверкой допустимости хода (например, невозможность прохода через стены).

Также была реализована базовая система взаимодействия с окружением, включая проверку столкновений.

#### 5. Добавление противников и базового ИИ
После реализации игрока были добавлены противники. Для них реализовано простое поведение, основанное на перемещении в сторону игрока при его обнаружении.

Была внедрена базовая логика искусственного интеллекта, включающая преследование и атаку при сближении.

#### 6. Реализация боевой системы
На данном этапе была разработана пошаговая боевая система. Реализованы механики нанесения и получения урона, проверка состояния здоровья персонажей, а также условия победы и поражения.

Боевая система интегрирована в общий игровой цикл, где каждое действие игрока сопровождается ответным действием противников.

#### 7. Добавление системы заклинаний
Одним из ключевых расширений проекта стала система заклинаний.

Были реализованы магические способности игрока, позволяющие влиять на игровой процесс. В систему вошли различные типы заклинаний:
- атакующие заклинания (нанесение урона по противникам);
- лечебные заклинания (восстановление здоровья игрока);
- усиливающие эффекты (временное повышение характеристик).

Система заклинаний добавила элемент стратегии, позволяя игроку выбирать оптимальный способ прохождения уровней в зависимости от ситуации.

#### 8. Интеграция игровых систем
На завершающем этапе все системы были объединены в единый игровой цикл. Обеспечена корректная работа взаимодействия между картой, игроком, противниками и системой заклинаний.

Также была проведена отладка механик, устранены ошибки взаимодействия и сбалансированы основные параметры игры.

---

### Итог реализации

В результате была разработана полноценная прототипная roguelike-игра с процедурной генерацией уровней, пошаговыми боями, системой противников и заклинаний.

Проект демонстрирует применение объектно-ориентированного подхода, работу с внешними библиотеками и понимание архитектуры игровых приложений. В дальнейшем игра может быть расширена и интегрирована в Telegram-бот как элемент геймификации образовательного процесса.
